Overview

In order to offer users an efficient user experience an intuitive interface, I addressed Harvard branding, consistency and familiar conventions, such as buttons, breadcrumbs, etc.

On this first draft, I reviewed the “Employee Self Service” landing page and “View Paychecks.” As I continue with the other pages, I’ll keep addressing consistency, so that the user would find a clear and familiar interface even accessing different content areas.

Labels, links and buttons


Avoid to use only icons. Make the function of buttons clear with text labels.

Current buttons and links

Proposed consistent buttons with clear labels and icons


Use primary buttons to help users to distinguish primary from secondary activities.
Define the position of buttons (beginning or end of the flow) based on common conventions and ideal user flow.

Ideal user flow, or "happy path", is the favorable and most direct outcome the interface is designed to provide in order to accomplish a task. In this case, users are expected to (1) review the available information (i.e.: home phone number), (2) decide what to do with it (edit or delete), (3) decide on alternative actions (add more information or move on.)

Current buttons

Proposed consistent buttons with clear labels and primary buttons

Use the same "happy path" criterion with buttons on other interfaces, such as modals.
Add text to form labels to clearly define this information (such as "Phone Number") and clearly distinguish a drop down menu from a text entry form.
Use input form placeholder to provide hints on the format to use for entering information correctly (and therefore avoiding error messages).
Vary the height of modals based on the information they contain, to avoid extra white space and possible user's assumption that information may be missing.

Employee Self Service

Here is a draft of the user interface recommendations for the home page. By making the cards and icons smaller there is a better distribution of the cards. By wrapping and centering the headers, making the icons smaller, and arranging the dynamic labels and values, helps with consistency and doesn’t compromise legibility.
I also substituted the Oracle logo with the Harvard logo, and changed the background colors to be consistent with the Harvard palette as documented in the guidelines.

I would suggest to position the “Classic Home” card to a secondary placement, since we obviously want the user to appreciate the new and more efficient content organization. If they go there the very first time, they may never bother with the other cards.
I would also propose (not depicted in this screen) that the cards would be arranged based on frequency of use, whether it is checking a biweekly pay check, or reviewing an yearly benefit package, biyearly performance activities, etc. A quick user testing would help us to learn the user perception of a more efficient order or even visual grouping of cards.

User Flow

The order and position of the cards should be determined by the frequency of use.
If possible, provide links to information of immediate need.

User research would reveal useful information, such as a more frequent use of the Learning and Development due to frequent logging of taken courses and training.

Personalize NavBar

Personalize NavBar should preserve common conventions and patterns, such as placing at the bottom of the interface the buttons that address the completion of the primary task.

The same icons shall not be used to represent different functions such as the upper nav menu "hamburger" icon, and the Personalize NavBar vertical sorting icon.
A vertically sorting function would be better represented with the equivalent to the Bootstrap glyphicon icon glyphicon-option-vertical.

Accessibility

Refer to the HUIT Online Accessibility website for making content and navigation accessible to users of assistive devices.

To facilitate the access to accessibility content to users of screenreader software, consider placing hidden links on the home page. Although these links are invisible to most users, they are easily accessible by screenreaders. This approach is described in this article - http://webaim.org/techniques/css/invisiblecontent/#skipnavlinks.
