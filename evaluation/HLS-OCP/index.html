Overview

This User Experience evaluation (Usability, Mobility and Accessibility) was not conducted with any user, including users of assistive technology. Once these issues listed here are resolved, it is highly recommended to test the product with target users, as well as with users of assistive technology.

While most of the information could be accessed by a computer, the initial issues encountered with a mobile browser prevented a mobile evaluation.

Select any of the images for a full scale view.

Accessibility Evaluation

Specifications

This accessibility audit was conducted on a MacBook Pro with a Macintosh operating system version 10.13.2.

The product was accessed with a Chrome browser via https://harvardtest.sabacloud.com/Saba/Web_spf/NA3T1SNB0035/common/ledetail/HLS-00013380

The testing tools used for this audit were:

  • Google Inspect for code inspection
  • Chrome Vox browser extension to simulate screen reader technology
  • WebAIM Color Contrast Checker to check the validity of the color contrast for visual impairment

While most of the information was accessible by a computer, the issues encountered with a mobile browser determined that a complete mobile evaluation cannot be completed until these issues are resolved.

Accessibility of information - ALT information

Finding Recommendation
Most images in color banners don’t have ALT tag information (see icon of folder next to the header “Protecting High Risk Confidential Information”). When a screen reader accesses them, it detects them only as “image”. Provide descriptions for all images, regardless if they are meaningful to the content or just decorations.
Finding Recommendation
The information accessed by a screen reader seems to refer to the filename of this image. Provide descriptions for all images, regardless if they are meaningful to the content or just decorations.

Information that is present both in the audio track and matched in the transcript, B. questions and C. navigation instructions throughout the presentation.
Finding Recommendation
Although readable in a linear fashion (one column at the time) table information should always be structured in a HTML table. Use HTML tables with all tabular information throughout the presentation. Never use HTML tables for page layout.
Finding Recommendation
Because the text slides used for this sequence advance automatically while the audio plays, it is not possible for a screen reader to access both the audio and textual information on the page. This information is different than the one in the transcript. This content is not displayed consistently with the rest of the presentation, where each new content is presented on new slides. These slides also don’t show the navigation dots. Separate each phrase and corresponding audio on separate slides.

Accessibility of information - Color contrast

Web Content Accessibility Guidelines 2.0 level AA requires a contrast ratio of 4.5:1 for normal text and 3:1 for large text. Large text is defined as 14 point (typically 18.66px) and bold or larger, or 18 point (typically 24px) or larger.

Navigation buttons of throughout the presentation.

Finding Recommendation
Start and Submit buttons throughout the presentation. The contrast ratio between the color of the type #FFFFFF and the background #35CBF2 is equal to 1.91:1. Change the background color from #35CBF2 to #027CAC to reach a contrast ratio above the required 4.5:1, or change the background color of the button to white #FFFFFF and the color of the font to a blue with sufficient contrast #027CAC.
Disabled Prev and Next buttons. The contrast ratio between the color of the type #838383 and the background #D6D6D6 is equal to 2.61:1 - Increase the font-size of the from 8 points to 12 points. Change the font color from #838383 to #5C5C5C to reach a contrast ratio above the required 4.5:1.
Finding Recommendation
Paragraph "Develop an understanding of the duties..." and all the other slides with this font size, font color and background color. The contrast ratio between the color of the type #FFFFFF and the background #02A1E0 is equal to 2.93:1. Increase the font-size of the from 8 points to 12 points. Change the background color from #02A1E0 to #027CAC to reach a contrast ratio above the required 4.5:1.
Finding Recommendation
The contrast ratio between the color of the type #FFFFFF and the background #2C9189 is equal to 3.81:1.. Increase the font-size of the from 8 points to 12 points. Change the background color from #2C9189 to #288179 to reach a contrast ratio above the required 4.5:1.
Finding Recommendation
The contrast ratio between the color of the subdued type in the left navigation #909090 and the background #F3F3F3 is equal to 2.88:1. Increase the font-size of the from 7 points to 12 points. Change the font color from #909090 to #6E6E6E.
Alternatively, change the font color to #757575 and the background color to #FFFFFF to reach a contrast ratio above the required 4.5:1.

While the second half of the audio track displays two new paragraphs and a button, the audio track becomes silent, although it still plays. The information from the audio track and its relationship with the information shown on the slides should be consistent throughout the presentation.
Finding Recommendation
Unless the cursor is moved over the image, there is no indication that the scrolling of this content can be interrupted. This is a useful and accessible feature but it should indicated on this interface. Show a caption with "Click on the content area to control the scrolling of the content" or in the audio and the transcript.
The functionality of the scrolling of this content via mouse click is unpredictable. Once the scrolling reaches the bottom of the document, it automatically scrolls back to the top. At this point several mouse clicks cannot initiate the scrolling again. When scrolling content, always show scroll bars to indicate the progress of this action.
Finding Recommendation
Text entry area doesn’t provide any boundary. An excess of text bleeds outside the text entry area and disables the functionality of the Submit button. Keep the text within a defined text area with a scrolling bar when needed.
While the Submit button doesn’t advance to the next slide until content is typed in the text entry area, the Next button is enabled. Provide consistency of behavior with all the interface elements. Here both the Submit and Next buttons should be disabled until text content is provided by the user.
Finding Recommendation

It is not apparent yet that the extra transcript belongs to a video that will be played later. The content of the transcript should match the instance of the content of each slide. This slide should only show the paragraph that matches the audio track. The audio track should contain content related to each slide. It shouldn’t be silent as in this slide.
Finding Recommendation
On this and other slides a portion of the audio track is silent. The audio track should contain content related to each slide. It shouldn’t be silent as in this slide.

Usability of information - Interface consistency

Finding Recommendation
While on some slides the header of the transcript matches the apparent header of the slide, on other slides (Pick One, Takeaways, etc.) either the transcript doesn’t match the title of the slide or the slide doesn’t show the tile of the transcript. The headings of transcripts and slides should match consistently throughout the presentation.
Finding Recommendation
In this page, the video is played by the user, while on previous slides the video starts automatically. Provide consistency of controls throughout the presentation.
Finding Recommendation
On the "Competence and Diligence / Scenario Summary" the selection of the Submit button, although enabled, doesn't provide any feedback. On "Pick One", the Submit button triggers a warning pop-up. Provide consistency of behavior throughout the presentation. Either disable the Submit button until the action requirements are met, or provide a warning. Because of potential accessibility issues and to simplify the user interface behavior, the former option is recommended.
Finding Recommendation
The selection of each item is replicated in the sentence above. The design of this interface doesn’t employ standard web interface elements like radio buttons. Use typical web interface conventions and an accessible layout throughout the interface.

Usability of information - Content display

Checkboxes and radio buttons have inconsistent visual clues from one summary to another. At times, they are highlighted on mouse-over, other when the wrong and correct answers are provided - provide consistent visual clues throughout the presentation.

Finding Recommendation
While the content of the first two tabs show the "Confidentiality of Information" header, the rest of the tabs don't show any header. This header should also refer to the selected link, for instance when selected "Rule 1.6(a)", the header of the correspondent paragraph should be “Rule 1.6(a) Confidentiality of Information”.
While there is no space prior to the parenthesis in "Rule 1.6(a)", there is a space in "Rule 1.6 (c)". 		Provide consistency of information (headings in the case) throughout the content of the same topic. Provide consistency of information display (character spacing in the case) throughout the presentation.
Finding Recommendation
The warning icons (yellow triangular) and the error icons (red "X") shown next to the wrong answers are confusing. Use only one and a consistent type of error icon throughout the presentation.
Although each radio button provides a different explanation which is useful to the user (after the Submit button is selected) the explanations for the other radio buttons cannot be accessed unless a user accesses this slide by going back, selecting and submitting. Allow for all the information to be accessed. In this case, allow for the selection and submittal of all the radio buttons without leaving this slide.
Finding Recommendation
The audio track scrolls without audio and the video doesn’t start automatically, nor there is the instruction to start the video. The Next button is active although the video hasn’t been played yet. The audio track should not be shown when it is not necessary. The video should either start automatically or instructions should instruct the user to play the video. The next button should either remain disable until the video is played or always enabled. These conventions should be consistent throughout the presentation.
Finding Recommendation
When the audio reads the second paragraph, the grey background appears a few seconds before the text. Provide consistent visual clues throughout the presentation.

Finding Recommendation
Under "Duty and Confidentiality", the left navigation lists 15 slides, while the first slide shows only 12 navigation dots. Allow for visual consistency between the left navigation and the slides. Either list all the 12 slides in the left navigation, or show 15 dots on each slide.
