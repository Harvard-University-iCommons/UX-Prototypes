Overview

A competitor analysis consists of knowing how to research properly, understanding exactly what information to looking for and synthesizing that information before acting on findings.
Competitor analyses help solve usability problems, help understand where the product or service stands in the market, inform the design process to know the strengths and weaknesses of the competition, to have reliable evidence when making product changes and to focus efforts in a target market. (Reference: https://usabilitygeek.com/how-to-do-ux-competitor-analysis/ )

This study examines the current Course Planner interface and features, as well as two popular calendar products: Google Calendar and Outlook Calendar. This process helps to inform recommendations for improvements of the Course Planner interface.

SEAS Course Planning - Features

The main purpose of EAS Course Planning is to show a weekly view of courses, which can be listed according to term selections and filtered according to undergraduate status and catalog prefix.

Course information

Course details (course catalog number and time) can be viewed upon mouse-over.

Upon mouse-over
  1. The course elements scheduled to begin at the same hour overlap the others courses
  2. Within the highlighted hour range, all the instances of the course "touched" by the mouse-over are made relevant. A stylized tooltip shows course details information

Annotated screen capture of the SEAS Course Planning interface at https://info.seas.harvard.edu/courses/#/schedule

Links and filters

Drop down options can either show different content or filter the displayed information.

  1. Selections from the "Semester" drop down change the information on the web page
  2. Selections from the "Undergraduate Status" and the "Catalog Prefix " drop downs filter the information on the web page

Annotated screen capture of the SEAS Course Planning interface at https://info.seas.harvard.edu/courses/#/schedule

Mobile

Although this interface could be easily viewed on a mobile device, it is not currently resoponsive.

  1. The SEAS Course Planning interface is not responsive, since it doesn't adjust to the mobile display and requires horizontal scrolling
  2. The selection of the "Semester" drop down is compromised by the "FOUR YEAR PLAN" link.

Screen captures of the SEAS Course Planning interface on a Samsung 9, Android OS, at https://info.seas.harvard.edu/courses/#/schedule

Google Calendar - Features

Google calendar events can viewed by day, week, month or year.

Interface

Upon mouse-over
  1. The interface displays of a number of events that share the same starting time
  2. Overlapping events don't compromise the indication of event length
  3. Current day is highlighted
  4. Interface layout allows for additional content

Annotated screen capture of the Google Calendar interface at https://www.google.com/calendar

Event details, filters and links

  1. Selections of available calendar time allow to create a new event
  2. Selections of an existing event shows the event name, time and location
  3. Each event can be viewed, edited, duplicated or deleted, in addition to other choices
  4. Navigation links view other time events
  5. Search for events by any related detail
  6. Filters show different time lengths and calendars
  7. The timeline matches the viewer's timezone

Annotated screen capture of the Google Calendar interface at https://www.google.com/calendar

Recommendations

The following interaction flows and wireframes propose usability improvements for the current Course Planner interface.

The effect of an event is represented by an interaction flow connection. The interaction flow expresses a change of state of the user interface. An event can also trigger an action, which is executed prior to updating the state of the user interface; for example, a delete or update operation on instances of a database.

Wireframes, also known as a page schematic or screen blueprint, are visual guides that represent the skeletal framework of a website. Wireframes are created for the purpose of arranging elements to best accomplish a particular purpose.
