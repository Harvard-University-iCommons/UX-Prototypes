Overview

This User Experience evaluation (Usability, Mobility and Accessibility) was not conducted with any user, including users of assistive technology. It is highly recommended to test the product with target users, as well as with users of assistive technology.

Select any of the images for a full scale view.

Accessibility Evaluation

Specifications

This accessibility audit was conducted on a MacBook Pro with a Macintosh operating system version 10.13.2.

The product was accessed with a Chrome browser via https://harvardtest.sabacloud.com/Saba/Web_spf/NA3T1SNB0035/common/ledetail/HLS-00013380

The testing tools used for this audit were:

  • VoiceOver to simulate screen reader technology
  • Google Inspect for code inspection
  • Chrome Vox browser extension to simulate screen reader technology
  • WebAIM Color Contrast Checker to check the validity of the color contrast for visual impairment

Summary

A cursory inspection of the content didn’t reveal any relevant accessibility issue.
Even though the content seems to be well within the A and AA WCAG 2.0 guidelines, only testing with native users of assistive technology will ensure that the content is properly accessible.

Usability Evaluation

Specifications

This usability audit was conducted on a MacBook Pro with a Macintosh operating system version 10.13.3.

The product was accessed with a Chrome browser via https://gsas.harvard.edu/admissions and https://gsas.harvard.edu/apply-non-degree-programs

The testing tool used for this audit was:

  • Google Inspect for code inspection

Usability of information - Information hierarchy

Finding Recommendation
The height allocated to the header text and image moves key information outside the viewable area. Allow for the most important information to be visible within the viewable area. The text and the image of the header should not compromise the visibility of more important information like the admission link.

Usability of information - Navigation

Finding Recommendation
It is not clear why the same link has to be selected multiple times. Avoid multiple selections of the same link unless the reason for doing so is justified and clearly stated.

Confirmation screen.

Finding Recommendation
Habitually primary buttons are located on the lower right hand corner of the content area. Maintain a constant position of buttons and avoid accidental selections by separating Cancel from other buttons.

Proposed buttons position.

Finding Recommendation
In order to facilitate the accessibility of the navigation, navigation buttons and links should be in close proximity. Although the left navigation bar allows users to navigate in both directions, allow also for a back button at the bottom of each page.

Proposed Previous button.

Usability of information - Links and buttons

Finding Recommendation
The function of the Delete link is not clear. Provide an explanation for the Delete link. Can forms be ignored instead?
Text links should be consistently used for content navigation (either linking to content in the same web page or to other web pages).
Buttons should fulfill a specific function, like submitting a user choice.
The Delete link should be a button.

Personal Background forms.

Usability of information - Content clarity

Finding Recommendation
Inappropriate information causes confusion. Remove residual programming terms and clues. In this case “apply.gsas.harvard.edu Says” is not needed.

Warning message.

Finding Recommendation
Unnecessary links cause confusion. Once deleted the “Permanent Address”, the "Same as permanent" link remains on the "Mailing Address" and on "New Address" content. Although users will learn that this link is not useful after reading a warning message, avoid confusion and the requirement of extra selections by removing the "Same as permanent" upon the deletion of the "Permanent Address."

Mobility Evaluation

Specifications

This mobility audit was conducted on a Samsung S8 and on Android operating system.

The product was accessed with the Samsung default mobile browser via https://apply.gsas.harvard.edu/apply

High level summary

Although the content is accessible with a mobile device, the touch target size is not sufficient. Each active element should be well within an area of between 44 x 44 pixels (iPhone standard) and 28 x 28 pixels (Nokia standard).
