Overview

Review of the Harvard Network Registration web site at getonline.harvard.edu.

Usability - extent to which a product can be used by specified users to achieve specified goals with effectiveness, efficiency and satisfaction in a specified context of use.

Accessibility - extent to which products, systems, services, environments and facilities can be used by people from a population with the widest range of user needs, characteristics and capabilities to achieve identified goals in identified contexts of use.

Responsive Design - Ra Web design approach aimed at crafting sites to provide an optimal viewing experience — easy reading and navigation with a minimum of resizing, panning, and scrolling — across a wide range of devices (from mobile phones to desktop computer monitors).

User Experience - a person's perceptions and responses that result from the use and/or anticipated use of a product, system or service.

Summary

  • Usability - Adjust the layout to make a better use of the vertical space. Avoid to the user to having to scroll in order to view information of primary importance
  • Accessibility - Ensure that each link contains contextual information (not just "here"). Use title tags to provide additional contextual information for each link.
  • Responsive Design - Use scalable fonts and vertical spacing to display more information on mobile displays.
  • User Experience - Enhance the information hierarchy and facilitate the user's understanding of the available choices with alternatives to the content layout.

Accessibility Review

Information Access

Accessibility Recommendations

Provide a description to links via the title tag.
When screen readers scan the content of an interface for links, they are programmed by the user to filter all the links, without the textual information around them.
Without title tags on each link, the links on this page would be presented as a series of 6 "here" links, which don't provide any cotextual information.
Either include descriptive information within <a></a> tags, or add that information to title tags.

Responsive Design Review

Responsive Design Recommendations

The Harvard's Network website is responsive on several and typical mobile displays.

Consider scalable, responsive font sizes, as recommended in this article - Use legible font sizes

Avoid to use br or empty p elements to control vertical space. Use proportional increments defined by CSS instead.

Following the above usability recommendations would ensure more visibility of the primary content, prior to scrolling to access the information not into view.
