Overview

This is a summary of the outcome of last week’s user testing sessions, as planned on the Task Worksheet and documented on the Task Observation Sheet.
The user experience findings documented here refer to the items listed in column A of the Task Observation Sheet

The participants to these user testing sessions where HUIT Support, but with great familiarity with users in need of technical help. Although this information is sufficient at this stage of this project, employing actual users on user testing (HUIT employees, faculty and students) would both validate this information and provide additional insight.

In the following findings, the term “users” refers to the individuals in need of technical help, while “Support” refers to the individuals providing that help.

Usability of the access to Service Now portal

The purpose and label of the links that access ServiceNow should be immediately clear.
For those users dealing with great urgency, provide also phone and email information to contact Support.

Task 1 Observations

Recommendation 1 - For those users dealing with great urgency, provide also phone and email information to contact Support.

The clarity of the purpose of the Service Now top navigation links

The difference between Submit a Request and Report an Issue is unclear.

Service Dashboard - This term is unclear. A community blog about problems experienced by users? Users feedback? System/security updates?

The purpose of the top navigation links should be recognizable.

Top navigation links

Task 3 Observations

Allow for tooltips to concisely describe the information behind each tab.

Current interface with no tooltips

 

Proposed interface with tooltips

Report an Issue

Although this interface is comprehensive and usable, most users want to act quickly when reporting a problem. Writing an email would be easier and faster than filling up these forms.
 

Service description and options

Task 4 Observations

Email and phone information should be shown in a prominent way for those users that would either want to contact Support because of an urgency, for guidance in providing this information, or for inquiring about an already submitted issue(s).

Current interface header

 

Proposed interface with contact information for IT support in each school.

 

Previous Incident

Task 4 Observations

A user usually doesn't remember a previously submitted ticket. A list of previously submitted tickets to select from should be listed here.

Current Step 2 - Previous Incident? and Previous Incident Number.

 

Proposed interface with list of previously submitted tickets to select from. The Previous Incident Number interface is nt longer needed.

Knowledge Base

Task 4 Observations

The usefulness of this information was inconsistent, although users couldn’t relate to the information to what was typed in the input form.
